Automation of Office Work at the CSE Department

Prepared by: S.Oyyathevan

Date: 23.01.2025

Version: 1.0

1. Introduction

This document outlines the requirements for automating the office work of the Computer Science and Engineering (CSE) Department...

2. Purpose

  • Define the functional and non-functional requirements for the automation system.
  • Provide a clear understanding of the system's scope and objectives.
  • Serve as a reference for developers, testers, and stakeholders.

3. Scope

Stores-Related Activities:

  • Material issue procedures.
  • Store ledger entry procedures.
  • Increasing indents on vendors.

Student Grading Activities:

  • Course registration.
  • Grade entry and management.
  • Generation of grade reports.

Student Leave Management Activities:

  • Leave application submission.
  • Leave approval workflow.
  • Leave record maintenance.

4. Functional Requirements

4.1 Stores Management Module

  • Material Issue
  • Store Ledger Entry
  • Indent Generation

4.2 Student Grading Module

  • Course Registration
  • Grade Entry
  • Grade Reports

4.3 Leave Management Module

  • Leave Application
  • Leave Approval Workflow
  • Leave Records

5. Non-Functional Requirements

  • Performance
  • Usability
  • Security
  • Scalability
  • Cost

6. System Architecture

The system will be a web-based application with components:

  • Frontend: HTML, CSS, JavaScript (React.js or Angular)
  • Backend: Python (Django/Flask) or Node.js
  • Database: MySQL or PostgreSQL
  • Hosting: AWS Free Tier or On-premise server

7. Data Flow Diagrams (DFDs)

7.1 Level 0 DFD (Context Diagram)

External Entities: Students, Faculty, Storekeeper, Clerks.

Data Stores: Student Database, Inventory Database, Leave Records.

7.2 Level 1 DFD

Processes: Stores Management, Student Grading, Leave Management.

Data Flows: Input forms, Output reports, Notifications.

8. User Interface Requirements

  • Role-based dashboards.
  • Digital forms for processes.
  • Printable reports.

9. Assumptions and Constraints

  • All users have basic computer literacy.
  • The system must be operational within 6 months.

10. Risks and Mitigation

  • Risk: Limited budget → Use open-source tools.
  • Risk: Resistance to change → Provide training.

11. Glossary

  • HoD: Head of Department
  • Indent: A formal request for supplies
  • GPA: Grade Point Average

12. References

  • Existing forms and documents from the CSE department.
  • Interviews with office staff, students, and faculty.
